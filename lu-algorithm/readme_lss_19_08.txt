Документация для программы.
Данная программа решает систему линейных уравнений с помощью метода LU разложения.

Описание программы:
1. Сначала считываем размерность и саму матрицу, а также вектор B.
2. Дальше проверяем матрицу А на вырожденность 
3. Строим матрицы L и U
4. Находим к ним обратные матрицы.
5. Применяем матричное умножение L*B=Y, а затем U*Y=X мы находим вектор X

Подробное описание вышеуказанных алгоритмов можно найти в книге К. Ю. Богачёв Практикум на ЭВМ. Методы решения линейных систем и нахождения собственных значений.

Функции, реализованные в файле lss_19_08.c:
    int lss_19_08(int n, double *A, double *B, double *X)
        Описание:
            Основная функция модуля, решающая СЛАУ методом LU.

        Возвращаемые значения:
            0 - решение построено
            -1 - метод не применим к данной матрице
      

    void MatrixObr (int n, double *Array, double *E);
        Описание:
            Для нахождения обратной матрицы через единичную матрицу

    int determinant(int n, double *Matrix);
         Описание:
            Для нахождения определителя матрицы, чтобы проверить на вырожденность. Используется метод Гауса.

    void Lxb(int n, double *L, double *B, double *Y);
        Описание:
            Для нахождения L*B=Y

    void Uxy(int n, double *U, double *Y, double *X);
        Описание:
            Для нахождения U*Y=X



Требования к памяти: Вычислительному модулю требуется n * n * sizeof(double) дополнительной памяти.
   
Применимость: Метод применим для обычных матриц с невырождеными главными минорами.

Использование программы:
    Usage: lss [input_file_name] [output_file_name] [options]
    Where options include:
     -d                print debug messages [default OFF]
     -e                print errors [default OFF]
     -p                print matrix [default OFF]
     -t                print execution time [default OFF]
     -h, -?            print this and exit
    Default input_file_name value is lss_00_00_in.txt, default output_file_name value is lss_00_00_out.txt.
Формат входных данных.

        n
        a_1_1 a_1_2 ... a_1_n
        a_2_1 a_2_2 ... a_2_n
        .....................
        a_n_1 a_n_2 ... a_n_n
        b_1
        b_2
        ...
        b_n

    Здесь n - число уравнений в системе (число неизвестных всегда считаем равным числу уравнений),
    a_ij - элементы матрицы системы, b_i - элементы вектора правой части.
    Формат чисел: n - "%d", a_ij и b_i - "%lf".

Формат выходных данных, если решение существует

        n
        x_1
        x_2
        ...
        x_n

    Если решения не существует, выходной файл должен содержать одну единственную цифру 0.
    Формат чисел: n - "%d", x_i - "%1.9lf".

Коды возвращаемых ошибок:
    -2 - Метод LU не применим к матрице с нулевым главным минором
    -1 - Метод LU не применим к несимметричной матрице
    0 - Программа успешно решила задачу. Решение построено
    1 - Программа успешно решила задачу, но решения системы не существует, программа вывела в файл '0'
    2 - Ошибка валидации параметров. Вы ввели больше двух названий файлов.
    3 - Ошибка валидации параметров. Введённого вами параметра не существует.
    4 - Ошибка валидации параметров. Порядок параметров и названий файлов неверный.
    5 - Ошибка валидации параметров. Файла для входных данных с таким названием не существует.
    6 - Введённый Вами файл для входных данных пуст.
    7 - Размер матрицы системы не является целым положительным числом.
    8 - Количество элементов матрицы меньше необходимого.
    9 - Один из элементов матрицы не является числом.
